## Work that's shipped

**[AI Engineer Assessment Platform](https://ai-assessment-platform-one.vercel.app)** — *live product*
A hiring platform that replaces LeetCode-style puzzles with something that actually predicts AI-engineering skill:
candidates debug a simulated RAG pipeline instead of writing code. Multi-tenant, JWT-secured, role-based access
for admin/reviewer/candidate. A deterministic engine models 4 distinct RAG failure modes, and every candidate
action is logged with monotonic timestamps for full session replay. Shipped end-to-end — Docker multi-stage
builds, GitHub Actions CI/CD, deployed on Vercel + Render.
`React` `TypeScript` `FastAPI` `PostgreSQL` `Docker` `JWT`

**[Schema Migration Gym](https://github.com/kansalshivam/schema-migration-gym)** — *Meta × Scaler × HuggingFace × PyTorch, Top 800 / 32,000+*
Built solo in a 48-hour finale in Bangalore, reviewed by Meta's engineering team. A Gymnasium-compatible RL
benchmark for SQL schema migration — 6 task types across 3 difficulty tiers, with adversarial cases specifically
designed to break greedy agents (zero-overlap renames, constraint-repair traps, structural ambiguity). Each
submission gets a fresh, isolated PostgreSQL container via the Docker SDK and is scored across 5 diff dimensions.
The result that got Meta's attention: heuristic agents score a perfect 1.0 on standard tasks but collapse to
0.48 on adversarial ones — hard evidence of exactly where rule-based systems stop working.
`Python` `FastAPI` `PostgreSQL` `Docker` `OpenEnv` `Gymnasium`

**[Pollution Response Dashboard](https://github.com/kansalshivam/pollution-navigation-system)** — *Delhi Govt Hackathon, Top 6 Finalist*
A planning dashboard built for municipal pollution-response deployment, taken from idea to finalist pitch.
`React` `TypeScript`

**[FOSSEE Workshop Booking](https://github.com/kansalshivam/fossee-workshop-booking)** — *open-source, IIT Bombay*
Mobile-first UI/UX overhaul of the FOSSEE workshop booking platform, shipped with progressive enhancement so it
